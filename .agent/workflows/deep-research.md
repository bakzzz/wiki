# Deep Research — Глубокое исследование

// turbo-all

Систематическое исследование любой темы с верификацией данных и структурированным отчётом.

## Шаги

1. **Уточнить тему у пользователя**
   - Что именно исследовать?
   - Какой контекст? (технология, рынок, сравнение, best practices)
   - Какой формат результата? (отчёт, сравнительная таблица, рекомендации)
   - Есть ли ограничения по времени/языку?

2. **Загрузить скил deep-research**
   ```
   Прочитать: ~/.agent/skills/deep-research/SKILL.md
   ```
   Следовать методологии из скила: 5 фаз, обязательные минимумы выборки.

3. **Фаза 1 — Research Plan**
   - Сформулировать 3-5 ключевых вопросов
   - Подготовить 5-7 поисковых запросов (EN + RU)
   - Определить категории источников
   - Показать план пользователю для подтверждения

4. **Фаза 2 — Сбор ресурсов (≥ 10)**
   ```
   search_web: "<тема> official documentation"
   search_web: "<тема> best practices 2024 2025"
   search_web: "<тема> site:reddit.com"
   search_web: "<тема> site:news.ycombinator.com"
   search_web: "<тема> comparison analysis"
   search_web: "<тема> site:medium.com OR site:dev.to"
   search_web: "<тема> github issues discussions"
   ```
   Для каждого ресурса: зафиксировать URL, тип, автор, дата, релевантность.

5. **Фаза 3 — Глубокий анализ**
   - Прочитать каждый ресурс через `read_url_content`
   - Извлечь ключевые тезисы, цифры, цитаты
   - Structured extraction: автор, методология, findings, limitations
   - Cross-reference: проверить факты в ≥ 2 источниках
   - Bias detection: проверить разнообразие источников

6. **Фаза 4 — Синтез**
   - Сводная таблица тезисов
   - Консенсус и разногласия
   - Тренды и рекомендации

7. **Фаза 5 — Отчёт**
   - Сгенерировать отчёт по шаблону из скила
   - Включить AI Disclosure
   - Заполнить таблицу метаданных

8. **Ревизия (обязательна)**
   - [ ] Ресурсы ≥ 10
   - [ ] Статьи ≥ 10
   - [ ] Комментарии ≥ 100
   - [ ] Офиц. документация ≥ 1
   - [ ] Cross-reference выполнен
   - [ ] Bias check пройден
   - [ ] Источники оформлены с URL и датой

   Если хотя бы один минимум не соблюдён → доискать до выполнения.

9. **Показать отчёт пользователю**
   - Предложить сохранить как артефакт
   - Спросить нужны ли дополнительные вопросы или уточнения
