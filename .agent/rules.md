# Agent Rules — Wiki

## Общие правила

- Язык общения — **русский**, если пользователь явно не переключается
- Все пути в ответах — **абсолютные**
- После изменений в структуре агента — **обновить** `registry.md`

## Создание сущностей

- Скилы — в `.agent/skills/`
- Воркфлоу — в `.agent/workflows/`

## Код и качество (Специфика проекта Wiki)

- **КРИТИЧЕСКОЕ ПРАВИЛО (Frontend):** Работать СТРОГО и ИСКЛЮЧИТЕЛЬНО в рамках **Ant Design 5.x**. Кастомный CSS (`.css`, `.scss`, Tailwind, styled-components) применять запрещено, если существует готовый компонент или дизайн-токен в Ant Design.
- **КРИТИЧЕСКОЕ ПРАВИЛО (Theming):** Все цвета, размеры, шрифты и стили задаются ТОЛЬКО через **дизайн-токены** в `theme.tsx`. В компонентах (App.tsx, AdminPanel.tsx и т.д.) запрещено hardcode-ить цвета (`#fff`, `#1677FF`, `background: '#F8FBFF'`). Вместо этого — `theme.useToken()` и чтение `token.colorBgContainer`, `token.colorPrimary` и т.д. Системный подход: изменение темы в одном месте (`theme.tsx`) автоматически обновляет всё приложение.
- **Backend (FastAPI):** Весь код должен быть типизирован, ответы строго проверяются через Pydantic.
- **Изоляция:** Учёт Tenant ID (RBAC) при любых запросах на чтение/запись обязателен.
