## Юзабилити

- [ ] Ключевые действия доступны за ≤3 клика
- [ ] Формы: inline validation, автозаполнение, маски ввода
- [ ] Loading states: skeleton/spinner для всех async операций
- [ ] Empty states информативны (не просто пустая страница)
- [ ] Error states понятны пользователю (не технические сообщения)
- [ ] Confirmation для деструктивных действий (удаление, отмена)
- [ ] Undo/redo для критичных операций
- [ ] Навигация: пользователь всегда знает где он (breadcrumbs, active state)
- [ ] Поиск работает предсказуемо (autocomplete, fuzzy matching)
- [ ] Mobile: нет hover-only интеракций, touch targets ≥44px

## Accessibility

- [ ] Семантический HTML (nav, main, article, section, aside)
- [ ] ARIA labels для интерактивных элементов без визуального текста
- [ ] Keyboard navigation: Tab order логичный, focus visible
- [ ] Контрастность текста: WCAG AA (≥4.5:1 для обычного текста)
- [ ] Alt text для информационных изображений
- [ ] Skip to content link
- [ ] Формы: label привязаны к input, error messages связаны aria-describedby
- [ ] Не только цветом: ошибки обозначены иконкой/текстом, не только красным
- [ ] Анимации: prefers-reduced-motion респект
- [ ] Screen reader тестирование (хотя бы VoiceOver / NVDA)

## Визуальная консистентность

- [ ] Design system / UI kit определён и используется
- [ ] Цветовая палитра: ≤5 основных цветов + оттенки, через CSS переменные
- [ ] Типографика: иерархия заголовков (h1-h6), ≤3 font-family
- [ ] Spacing system: фиксированная шкала (4px/8px базис)
- [ ] Единые border-radius, shadows, transitions
- [ ] Иконки из одного набора (не микс разных style)
- [ ] Dark mode: не инвертирование, а продуманная альтернативная палитра
- [ ] Responsive: компоненты перестраиваются, а не просто сжимаются

## Микро-взаимодействия

- [ ] Hover/focus эффекты для интерактивных элементов
- [ ] Transitions для state changes (200-300ms, ease-out)
- [ ] Feedback при клике (ripple, scale, цвет)
- [ ] Progress indicators для длительных операций
- [ ] Toast/snackbar для подтверждения действий
- [ ] Smooth scroll для anchor-навигации
- [ ] Skeleton loading вместо spinner где возможно

## Информационная архитектура

- [ ] Навигация: ≤7 элементов в главном меню
- [ ] Иерархия контента визуально выражена (размер, вес, цвет)
- [ ] CTA (call-to-action) визуально выделен на каждой странице
- [ ] Консистентные паттерны: одинаковые действия выглядят одинаково
- [ ] Onboarding: новый пользователь понимает куда двигаться
- [ ] 404/error pages с навигацией обратно
- [ ] Breadcrumbs или back-navigation для глубокий иерархий
